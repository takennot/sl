Sup, this is a documentation for the Quiz project and my personal thoughts about it.

Separation of Files: As requirement demands, everything is in separate files - html (structure), css (styling) & js (the actual logic)

Quiz Logic: The javascript file (quiz.js) contains all of the quiz related logic. It has implemented functions for setting up the questions, processing user clicks, providing feedback, and handling quiz finish. It uses canvas to render and adds a graphical aspect/side to the quiz, enhancing the user experience
(though, in my case users were probably horrified by the background image xD). Ah yeah and the restart functionality allows users to replay the quiz without refreshing the page (as one of the requirements demanded that).

Styling stuff: For the fancy look, I played around with the css file (style.css). The background image is glorious and you can not convince me otherwise. Like who doesn’t love a crackhead looking Mike Wazowski staring at them while they are taking the quiz?

Portfolio integration: There are 2 buttons to go back on the quiz page - one uses a js and other one is just a link. The js one basically just takes the current tab’s history (within a session) and calls the “back()” method. And the back method triggers the browser to go back to the previous page in the tab session history, simulating the effect of clicking the browser's back button (basically). The link one is… just a link, not much to say here xd. On the portfolio itself, the link to the quiz is on the nav menu on top.

My group: As you can probably see, the group consists of me, me & me. I worked alone because I thought it would be faster and more convenient.

Challenges:

I don’t have a lot of experience with using canvas so I felt like a caveman sometimes during the development. But I think it went fine and now I have more experience with it.

To summarise, the project was pretty fun and taught me some neat stuff.
